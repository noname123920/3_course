1. Что такое инкапсуляция? Модификаторы public и private.

Инкапсуляция — это один из основных принципов объектно-ориентированного программирования (ООП), который заключается в объединении данных (полей) и методов (функций), работающих с этими данными, в единый объект (класс), а также в сокрытии деталей реализации от внешнего мира. 

Цель инкапсуляции — защитить данные от прямого несанкционированного доступа извне и обеспечить их изменение только через строго определенные методы класса, что позволяет контролировать состояние объекта и предотвращать ошибки.

Модификаторы доступа (public, private, а также protected) используются для управления видимостью членов класса (данных и методов):

public: Члены класса, объявленные как public, доступны из любого места программы, в том числе извне класса. Это обеспечивает интерфейс взаимодействия с объектом.

private: Члены класса, объявленные как private, доступны только внутри этого же класса. Они скрыты от внешнего доступа, что является ключевым элементом сокрытия данных при инкапсуляции. 

2. Основные детали создания классов в С++: данные-члены (поля) и функции-члены (методы). Отличие class от struct.

Классы в C++ — это шаблоны для создания объектов, объединяющие в себе:
Данные-члены (поля): Переменные, которые хранят состояние объекта (например, int age, string name).
Функции-члены (методы): Функции, которые определяют поведение объекта и предоставляют контролируемый доступ к его данным (например, setName(), getAge()).

Отличие class от struct в C++:
В C++ class и struct функционально почти идентичны, оба могут содержать данные и методы. Основное различие заключается в модификаторе доступа по умолчанию:

В class члены по умолчанию являются private.

В struct члены по умолчанию являются public. 

Исторически struct использовался для простых структур данных, совместимых с языком Си, а class — для полноценных объектов ООП. Сегодня выбор между ними часто зависит от стиля кодирования: для простых агрегатов данных без сложной логики используют struct, для полноценных объектов с инкапсуляцией — class.

3. Что такое конструктор? Какие виды конструкторов есть в C++?

Конструктор — это специальная функция-член класса, которая автоматически вызывается при создании нового экземпляра (объекта) этого класса. Его основная задача — инициализировать данные-члены объекта и выделить необходимые ресурсы. Конструктор имеет то же имя, что и класс, и не имеет возвращаемого типа. 

В C++ существуют следующие виды конструкторов:

Конструктор по умолчанию (Default Constructor): Конструктор, который не принимает аргументов. Если программист не определяет ни одного конструктора, компилятор C++ автоматически создает простой конструктор по умолчанию.

Параметризированный конструктор (Parameterized Constructor): Конструктор, который принимает один или несколько параметров. Используется для инициализации объекта определенными значениями, переданными при его создании.

Конструктор копирования (Copy Constructor): Специальный конструктор, который создает новый объект как копию уже существующего объекта того же класса. Он вызывается, например, когда объект передается в функцию по значению или возвращается из функции по значению.

Конструктор перемещения (Move Constructor): (Появился в C++11). Используется для "перемещения" ресурсов (например, динамически выделенной памяти) из временного (rvalue) объекта в новый, более эффективно, чем копирование.

Список инициализации членов (Member Initializer List): Хотя это не отдельный вид конструктора, это синтаксическая конструкция, используемая внутри конструктора для прямой инициализации членов класса до выполнения тела конструктора, что является предпочтительной практикой в C++.
